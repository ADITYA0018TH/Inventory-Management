### 2. Internship Related Information

| Criterion | Remarks |
|---|---|
| **Constraints** | Designing a system that is both operationally flexible and regulatory-compliant was challenging. Ensuring real-time data consistency across 17 MongoDB collections and managing Socket.IO connections with a stateless REST API required careful handling. |
| **Understanding of the Issue** | Pharmaceutical organizations rely on manual spreadsheets and disconnected systems that lack real-time visibility and batch traceability. This causes stockouts, expired product losses, and compliance failures. A single integrated web platform is needed to automate these workflows. |
| **Likely Problems / Research Limitations** | MongoDB lacks relational constraints, requiring application-level validation. Socket.IO user targeting needs careful reconnection handling. The compliance formula may not cover all regulatory scenarios. IoT integration and advanced ML forecasting are out of scope for this version. |

| **Aim** | To develop a full-stack web application that digitizes pharmaceutical supply chain operations with real-time visibility, automated compliance, and end-to-end batch traceability. |
